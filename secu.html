⚙️ UC 1: MÉCANISMES DE SÉCURITÉ

Mécanisme Proposé Justification
Authentification Multifacteur Réduit le risque d'accès non autorisé
Contrôle d'accès RBAC Limite l'accès selon les rôles des utilisateurs
Isolation Réseau Empêche les connexions non autorisées
Chiffrement (repos/transit) Évite les interceptions et lectures par des tiers
Sécurité de l'Information Surveillance continue et alertes

🚨 UC 2: INDISPONIBILITÉ DES SERVICES API

⚠️Risques Identifiés

Panne/indisponibilité des APIs critiques
Attaques de déni de service (DoS/DDoS)
Mauvaise configuration réseau
Maintenance non tracée provoquant erreurs

🔍Mesures de Traçabilité

Journalisation via SIEM
Audit trail obligatoire (Bastion/console)
Comptes personnels + MFA administrateurs
Just-in-Time Access avec justification

🛠️ UC 2: SOLUTIONS ET CONTRAINTES

🔧Composants de Sécurité

Load Balancer: Redondance et gestion trafic
WAF: Filtrage attaques sur API
IAM + MFA: Authentification forte
Monitoring: Surveillance état API
Bastion Host: Accès sécurisé serveurs

⚖️Contraintes Opérationnelles

WAF: Risque faux positifs
Bastion: Complexification accès
SIEM: Volume logs important
Monitoring: Configuration fine nécessaire
